# Roadmap

Short-term direction for **RAYS-CORE-CLI** (RAYS-CORE on PyPI). Priorities evolve with feedback.

## Near term

- [ ] Expand automated tests beyond smoke/compile checks (mocked LLM fixtures for pipeline scenarios).
- [ ] Optional **`ruff`** / **`mypy`** in CI once noise is manageable.
- [ ] Trusted publishing from GitHub Releases to PyPI (OIDC), alongside manual uploads if needed.

## Medium term

- [ ] Dedicated troubleshooting section growth from real issues filed on GitHub.
- [ ] Optional `src/` layout if the codebase keeps growing.

## Contributing

Ideas welcome via [Issues](https://github.com/markknoffler/RAYS-CORE-CLI/issues); see **CONTRIBUTING.md** for branching and CI expectations after you push.
